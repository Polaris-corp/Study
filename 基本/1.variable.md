# 変数
#### プログラムを書く上で呆れるほど使用する「変数」について説明する

最初に出会う概念で、呆れるほど使用する。

というか、使用しないでプログラムを書くのはとってもつらそうだし、想像するだけで心拍数が上がってきそう

そこまで言わせる**変数**とは何者かという事を少しだけ説明しておきたい。


C#では、変数は「型」と言われるものと不可分なので、型についてもある程度は一緒に説明しておく。

### よく言われるイメージは箱

箱と言われても最初はピンと来ないと思うし、正直、呆れるほど書くので、忘れることもなくなるし、これも一回読んだら二度読む必要はないと思う。

---
とりあえず、変数がないと困るのってどんな時だろう

#### 変数で受け止められない世界

例えば、入力された二つの数字を足し算をするプログラムを書きたい時、

変数がない世界では、入力してEnterを押した瞬間にその入力値はソースコードの中を自由落下して墜落して消える。

かくして二つの数を足して表示したいという願いは届かなかったのである。**おしまい**。

       3と4が入力され、足し算した7を表示したい思いを載せた変数のないソースコード
       { 
         入力された3と4(受け止めるところがないのでどんどん落ちていく感じある)
            ↓　↓ 
            ↓　↓ 　
            3　↓
            ↓　↓
            ↓　↓
            ↓　↓
            ↓　4　
            ↓　↓
            ↓　↓
       } ←  ここまで来たら消える

↑**仮コード**、これをだんだんとC#のコードに変換(翻訳)していこうという話です。

#### 変数で受け止められる世界

この落下物達を受け止めて、自分の自由なところで使えるように出来ちゃうのが変数の良いところ

       3と4が入力され、足し算した7を表示したい思いを載せた変数のあるソースコード
       { 
         入力された3と4
            ↓　↓ 
            ↓　↓ 　
        変数 three ← input 3を受け止める用の変数
            ↓　↓
            ↓　↓
        変数 four ← input 4を受け止める用の変数
            ↓　↓
            ↓　↓
            ↓　↓
       } 変数に入っていようがここまで来たら消える

上の例でのthreeとfourが噂の**変数**である。

どれがそうなのかは変数って書いてあるから多分分かってくれたと思う。

ちなみに、ソースコードは上から下に流れていくのが基本なので、変数を使おうと思ったら、

**値を入れたい場所**か、**使いたい場所より上の場所**に一度書いておかないといけない

とりあえず書いておく事を変数の「**宣言**」って言ったりする。

でも、使いたいところの下に宣言しちゃっても大丈夫、VisualStudioでC#を書くのならソースコード真っ赤にしてここ直せって言われるので、仰せのままに直せばよろしい

--- 

##### 変数に受け止められた値は自由に取り出して使う事が出来る

とりあえず、宣言された後の変数は、自由に使用することが出来る

このコードなら、変数を宣言して、3と4を受け止めた後に、3+4を表示するような書き方をしてあげればよい

例によって仮コードに**3+4の表示処理**を追加しておく、値は[変数 three]と[変数 four]に入っている状態であるため、

3+4はthree + fourと書いておく

       3と4を入力したので7と表示したい思いを載せた変数のあるソースコード
       {
        入力された3と4
            ↓　↓ 
            ↓　↓ 　
        変数 three ← input 3を受け止める用の変数
            ↓　↓
            ↓　↓
        変数 four ← input 4を受け止める用の変数
            ↓　↓
        three + four を表示する
            ↓　↓
       } ここまで来たら消えるのは変わらない

これで**3と4が入力されて、3+4を表示する**という一連の流れは出来た。

##### 変数の宣言の書き方

変数の宣言は決まった書き方があって、それを覚える必要がある。

とは言っても大して難しくないよ！

        変数の型 変数の名前

という形で書けば良いだけ、簡単だね！

---
##### C#は変数を宣言する時に「型」と言うものを意識する必要がある

ここで、上の方と、この直前でちょっとだけ登場した「**型**」とか言うものを説明しよう

型にはまったとか、何々の型を取るとか、そんな意味で使用される事があるけど、

プログラムにおける型もイメージとしてはそんな感じで、

宣言された変数には型が合っている物(値とかよく言われてる)しか入らない。

上の例なら3と4は数字なので、数字用の変数を用意してあげる必要がある、

C#には数字用の変数の代表としてinteger（整数という意味）、略して**int**と呼ばれる型があって、

それをありがたく使わせてもらおう

        intを使って変数を宣言してみた

        int three

        int four

intを使用してthreeとfourを宣言したので、この変数は整数を扱うことが出来るようになりました。

他にも色々な型があるけど、とりあえず覚えておいてほしいのは

- **文字列**用の型として使用される、**string**
  - 文字列っていうのは単純に文字を一つなぎにしたものをイメージしてくれれば良いかな
    - (昔ヨーヨーで遊んでたのでストリングって聞いたらまず紐がイメージされました)
- **OK⇔NG**とか**はい⇔いいえ**を表現出来る、**bool**
  - https://ejje.weblio.jp/content/bool
    - ここを見ると、**主な意味:とてもいい；かっこいい** って出て来てよくわからないね
  - 略さないとbooleanなのでこっちでも調べてみよう
    - https://ejje.weblio.jp/content/boolean
    - **主な意味：ブール** どっちにしろイメージしにくいね
  - あなたは日本人ですか？→**はい！(bool型)**
  - あなたは僕のためにコンビニでパンを買ってきてくれますか？→**はい！(bool型)**
  - え？インフルなのに来ちゃったの？→**はい！(bool型)**
    - みたいな感じで使います、多少イメージが伝わったらいいなと思います

##### 本当に3と4だけしか入らないの？

上で使われたint threeとint fourは残念ながらここで死んでもらいます

例えば5と10が入力されたとしても変数名はint three、int four

1と9が入力されたとしても変数名はint three、int four

名前と変数に入っている値の関係歪みすぎですね？

ということで、色んな数字を受け止められるようにしてあげよう

入力された値を足し算したいので、とりあえずですが「左 ＋ 右 = 答え」みたいなイメージで

int left,int rightに変更でーす

       数字を２つ入力したので合計した値を表示したい思いを載せた変数のあるソースコード
       {
        入力された２つの数字
            ↓　↓ 
            ↓　↓ 　
        int left ← input 1つめの数字を受け止める用の変数
            ↓　↓
            ↓　↓
        int right ← input 2つめの数字を受け止める用の変数
            ↓　↓
        left + right を表示する
            ↓　↓
       } ここまで来たら消えるのは変わらない

##### 最終的に完成したコード

        void main()
        {
            // 文字列の変数に数字入力を受け取る
            // 入力形式は 数字スペース数字 である
            // 入力例1. 3 4
            // 入力例2. 10 5 
            // 等
            // 上のような形式でコンソールに数字を入力しEnterを押す
            // すると入力された内容がそのまま下の変数inputに入る
            string input = Console.ReadLine();

            // スペース区切りの数字が入力されてinputに入った、
            // 次はこのスペース区切りの数字をそれぞれ別々の変数に入れたい
            // ここで、「スペース毎に文字列を分解する」という処理をしてあげる
            // 分解したら下に文字列の配列numsに入れてあげる
            string nums[] = input.Split(' ');            
           
            // 分解した文字列から二つ、入力されてきた数字を数字型に変換して取り出してあげる
            int left =　int.parse(nums[0]);
            int right = int.parse(nums[1]);
            
            // 数字型の変数に入った数字を使用し、足し算した値をコンソールに表示してあげる
            Console.WriteLine(left + right);
        }







Tips:ちなみに変数が多くなると、処理が分岐したり、繰り返したりするよりも頭をより使うプログラムになるとかなんとか
